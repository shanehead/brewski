## Task 1: DB Migration + Complete Rust Rename

**Context:** The DB column and all Rust code currently uses `whirlpool_temp_c`. All of these must change atomically because SeaORM maps entity field names to DB column names — if the DB column is renamed but the entity field isn't (or vice versa), tests will fail. `models.gen.rs` is a generated file but we edit it manually here; Task 2 will confirm codegen produces the same result.

**IMPORTANT:** Do NOT run `just gen-entities` at any point — it regenerates entity files with wrong types (`Decimal` instead of `f64`) and causes ~280 compile errors. Only manually edit entity files.
