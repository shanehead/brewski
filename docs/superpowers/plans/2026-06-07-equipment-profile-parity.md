# Equipment Profile Parity Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add 11 new fields to equipment profiles to reach parity with Brewfather, switching boil-off input from %/hr to vol/hr and replacing the tun weight + specific heat fields with a single heat capacity field.

**Architecture:** SQLite migration adds/drops columns → codegen regenerates entity and model types from DB + OpenAPI → manual updates to TryFrom impl, calculations, and repository → frontend modal gains new inputs. The OpenAPI schemas are the source of truth for the Rust and TypeScript model types; always update those first and run `just gen-rust` / `just gen-ts` to regenerate.

**Tech Stack:** SQLite (SeaORM migrations), Rust/Tauri, OpenAPI (redocly), TypeScript, Svelte 5

ALTER TABLE equipment_profiles ADD COLUMN evap_rate_l_hr REAL NOT NULL DEFAULT 3.8;

  SET evap_rate_l_hr = evap_rate_pct_hr / 100.0 * boil_size_l;

```rust
/// Returns (pre_boil_volume_l, post_boil_volume_l, total_water_needed_l)
/// - pre_boil: cold volume needed in the kettle at start of boil
/// - post_boil: cold volume in kettle after boil
/// - total_water: pre_boil + hlt_deadspace (all water you need to start with)
#[allow(clippy::too_many_arguments)]
pub fn calculate_boil_volumes(
    batch_size_l: f64,
    boil_time_min: f64,
    evap_rate_l_hr: f64,
    trub_chiller_loss_l: f64,
    fermenter_loss_l: f64,
    top_up_water_l: f64,
    mash_tun_loss_l: f64,
    hlt_deadspace_l: f64,
) -> (f64, f64, f64) {
    let post_boil_volume = batch_size_l + trub_chiller_loss_l + fermenter_loss_l - top_up_water_l;
    let boil_hours = boil_time_min / 60.0;
    let pre_boil_volume = post_boil_volume + evap_rate_l_hr * boil_hours + mash_tun_loss_l;
    let total_water_needed = pre_boil_volume + hlt_deadspace_l;
    (pre_boil_volume, post_boil_volume, total_water_needed)
}
```
