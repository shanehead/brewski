# Batches & Recipe Versioning Infrastructure — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the Batches module with full recipe version snapshotting so users can record brew sessions against stable, queryable recipe snapshots.

**Architecture:** Recipe versions are created (or reused if unchanged) when a batch is created. A `RecipeVersionRepository` owns the dedup-and-snapshot logic. The Batch module follows the existing repository → command → api.ts → Svelte pattern throughout.

**Tech Stack:** SeaORM + sqlx (SQLite), Rust, SvelteKit 5 (Svelte runes), Tailwind CSS v4, OpenAPI codegen via `just gen`, entity codegen via `just gen-entities`.

        let recipe = RecipeRepository::new(self.db).get(recipe_id).await?;

            .filter(recipe_versions::Column::RecipeId.eq(recipe_id))
