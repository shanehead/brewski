# Mobile Support Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add iOS/Android support to Brewski via Tauri 2 mobile targets, with clean build-time platform separation using a Vite `$platform` alias — no runtime branching, no dead code shipped.

**Architecture:** A Vite alias resolves `$platform` to either `src/lib/mobile/` or `src/lib/desktop/` at build time based on `TAURI_ENV_PLATFORM`. Route pages become thin data-loading shells that import layout from `$platform`. Each platform directory contains complete, standalone implementations with no `{#if}` branching between platforms.

**Tech Stack:** Tauri 2, SvelteKit 5, Svelte 5 runes, Vite 8, Tailwind CSS v4, TypeScript 6.

import { defineConfig } from "vite";
import { sveltekit } from "@sveltejs/kit/vite";
import tailwindcss from "@tailwindcss/vite";
import path from "path";

const platform = process.env.TAURI_ENV_PLATFORM ?? "desktop";
