# Brewski Docs Site Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a VitePress documentation site in `/docs/site`, covering all Brewski features in a friendly, colloquial tone, deployed to GitHub Pages on push to `main`.

**Architecture:** VitePress lives in the `/docs/site` subfolder of the main repo (keeping it separate from the internal specs/plans in `/docs/superpowers`). GitHub Actions builds and deploys to GitHub Pages. Content is written as Markdown with a consistent structure: Getting Started → Guides → Reference → Concepts & Glossary → FAQ.

**Tech Stack:** VitePress, bun, GitHub Actions, GitHub Pages

**Style guide (follow precisely when writing content):**
- Write directly to the brewer. Use "you" and "your."
- Use contractions: "you're", "it's", "don't", "here's"
- Bold UI elements: **Add Hop**, **Save**. Use `→` for nav paths.
- Explain the *why*, not just the *how*.
- Avoid overly long sentences. If a sentence needs a comma and a semicolon, split it in two.
- No em-dashes. Use a comma, a period, or a new sentence instead.
- No "It is important to note that…", no "Navigate to X in order to…", no passive voice.

## Task 2: GitHub Actions deployment to GitHub Pages

**Files:**
- Create: `.github/workflows/docs.yml`

- [ ] **Step 1: Create the docs deployment workflow**

Create `.github/workflows/docs.yml`:

```yaml
name: Deploy Docs

on:
  push:
    branches: [main]
  workflow_dispatch:

permissions:
  contents: read
  pages: write
  id-token: write

concurrency:
  group: pages
  cancel-in-progress: false

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0

      - uses: oven-sh/setup-bun@v2

      - name: Install dependencies
        run: bun install

      - name: Build docs
        run: bun run docs:build

      - name: Upload artifact
        uses: actions/upload-pages-artifact@v3
        with:
          path: docs/site/.vitepress/dist

  deploy:
    environment:
      name: github-pages
      url: ${{ steps.deployment.outputs.page_url }}
    needs: build
    runs-on: ubuntu-latest
    steps:
      - name: Deploy to GitHub Pages
        id: deployment
        uses: actions/deploy-pages@v4
```

- [ ] **Step 1: Write `what-is-brewski.md`**

Content to cover:
- Brewski is a free, open-source homebrewing app for desktop (Mac, Windows, Linux). No subscription, no account required.
- Designed for homebrewers who want a Brewfather-style experience without the monthly fee or the cloud dependency.
- Key things you can do: design recipes from scratch, track batches from brew day to packaging, calculate ABV/IBU/SRM/water chemistry, import and export BeerXML.
- Your data stays on your machine. Cloud sync is optional and uses your own storage (Google Drive, iCloud, Dropbox).
- Brewski is opinionated about one thing: it should be fast, offline-first, and simple to use. No subscription tiers, no feature gates.

- [ ] **Step 2: Write `installation.md`**

Content to cover:
- Download the latest release from the GitHub releases page (link to `https://github.com/shanehead/brewski/releases`).
- macOS: `.dmg` file. Open it, drag Brewski to Applications. First launch may require right-click → Open if macOS blocks it.
- Windows: `.msi` installer. Run it and follow the prompts.
- Linux: `.AppImage`. Make it executable (`chmod +x`), double-click or run from terminal.
- First launch: Brewski creates a local SQLite database in your app data folder. No setup required.
- Where your data lives (brief note, with a link to Settings → Database Location for details).

- [ ] **Step 3: Write `first-recipe.md`**

Content to cover:
- Overview: a recipe in Brewski has ingredients (fermentables, hops, yeast, water), a mash, and an equipment profile. The stats sidebar shows OG, FG, ABV, IBU, and SRM as you build.
- Step 1: Hit **New Recipe** in the left sidebar. Give it a name and pick a style if you want.
- Step 2: Open the **Fermentables** tab. Hit **Add Fermentable**, search for your base malt (e.g. "Pale Malt 2-Row"), set the amount. Repeat for any specialty malts.
- Step 3: Open the **Hops** tab. Hit **Add Hop**, pick your variety, set the alpha %, use type (Boil, Dry Hop, etc.), and time.
- Step 4: Open the **Yeast** tab. Add your yeast and set the attenuation %.
- Step 5: Open the **Mash** tab. The default single-infusion mash step is already there. Adjust the temperature and water ratio.
- Step 6: Watch the stats sidebar. OG, ABV, IBU, and SRM update in real time. When you're happy, hit **Save**.
- Tip: You don't have to fill in everything at once. Save early and come back.

- [ ] **Step 4: Write `first-batch.md`**

Content to cover:
- A batch is what happens when you actually brew a recipe. It tracks the real numbers: your actual volumes, gravity readings, and carbonation.
- From a recipe, hit **New Batch**. Brewski links the batch to the current version of the recipe.
- Brew day tab: log your actual pre-boil and post-boil volumes and gravities. These aren't required — log what you measured.
- Fermentation: add gravity readings over time. Set the date, enter your gravity (in SG, Plato, or Brix depending on your settings), and optionally note the temperature.
- Carbonation tab: once you're ready to package, use the calculator to figure out priming sugar amounts or keg pressure.
- When the beer is done: mark the batch status as "Packaged" or "Complete." Add tasting notes, a rating.
- Your batch stays linked to the recipe version you brewed. If you update the recipe later, the batch still shows you what you actually brewed.

- [ ] **Step 5: Write `importing.md`**

Content to cover:
- Brewski supports BeerXML 1.0, which is the format used by Brewfather, BeerSmith, BrewUnited, and most other homebrewing tools.
- To import: hit the **Import** button in the recipe list sidebar (or go to File → Import). Pick your `.xml` file.
- What gets imported: recipe name, style, batch size, fermentables (name, yield, color), hops (name, alpha, use, time), yeast (name, attenuation), mash steps.
- What might not survive the import: custom ingredient data that isn't in your Brewski library, water chemistry details, equipment profiles (you'll need to pick one after importing).
- After importing, the recipe appears in your list. Review it — some values may need a quick adjustment if the source app stored things differently.
- Tip: Brewski matches imported ingredients by name. If a hop or fermentable doesn't match anything in the library, it's added as a custom entry. You can always edit it later.

- [ ] **Step 1: Write `building-a-recipe.md`**

Content to cover:
- A recipe is made up of: basic info (name, style, batch size, boil time), ingredients (fermentables, hops, yeast, water adjustments), a mash profile, and an equipment profile.
- The stats sidebar shows OG, FG, ABV, IBU, SRM, and pre-boil gravity. They update every time you change an ingredient.
- Walk through the tab structure: Fermentables → Hops → Yeast → Misc → Water → Mash → Notes. You don't have to fill them all in.
- Equipment profile affects your pre-boil volume, boil-off rate, and trub/fermenter losses. If you haven't set one up yet, Brewski uses sensible defaults.
- How to save: **Save** button in the header. Brewski tracks recipe versions — every time you save, you can see the history.
- The **Baseline Recipes** section in the sidebar has pre-built recipes you can clone and modify. Great starting point.

- [ ] **Step 2: Write `fermentables.md`**

Content to cover:
- Fermentables are everything that adds fermentable sugar to your wort: base malts, specialty malts, adjuncts (corn, rice, sugar), and extracts.
- Adding a fermentable: hit **Add Fermentable** on the Fermentables tab. Search by name. The library has hundreds of common fermentables.
- Key fields: Amount (kg or lb depending on your settings), Yield % (how much sugar this fermentable contributes per unit), Color (in Lovibond), Type (grain, liquid extract, dry extract, sugar, adjunct).
- "Add after boil" checkbox: for sugars and adjuncts added at flameout or later — these don't go into your mash efficiency calculation.
- The stats sidebar shows OG and SRM updating as you add fermentables.
- Creating a custom fermentable: if your ingredient isn't in the library, use the **Ingredients** section to add it. Set the yield % and color, and it'll be available in all your recipes.
- Tip: your base malt is usually 60–80% of the grain bill. Everything else is specialty.

- [ ] **Step 3: Write `hops.md`**

Content to cover:
- Hops add bitterness, aroma, and flavor. How much of each depends on *when* and *how* you add them.
- Adding a hop: hit **Add Hop**, search for a variety, set the amount, alpha %, use type, and time.
- Use types explained:
  - **Boil**: added during the boil. The longer the boil time, the more bitterness (IBU) you get, but less aroma.
  - **Whirlpool / Flameout**: added after the boil. Contributes flavor and aroma more than bitterness.
  - **Hopstand**: added during a hot steep after flameout. Brewski lets you set the hopstand temperature, which affects IBU contribution.
  - **Dry Hop**: added during fermentation or conditioning. Zero IBU contribution — it's all aroma.
- Alpha %: the bitterness potential of the hop. A higher alpha % means less weight needed to hit your IBU target. Don't skip this — it directly affects your calculated IBUs.
- The stats sidebar shows your total IBUs and the BU:GU ratio (bitterness to gravity ratio) in real time.
- Hop forms: pellets are the default. If you're using leaf/whole hops or cryo hops, set the form. It affects utilization. See [Hop forms](/concepts/hop-forms) for details.

- [ ] **Step 4: Write `yeast.md`**

Content to cover:
- Brewski uses yeast primarily to calculate estimated FG and ABV. The key field is **attenuation %**.
- Adding yeast: hit **Add Yeast**, search for a strain (e.g. "US-05", "WY1056", "EC-1118"). The library includes strains from most major labs.
- Attenuation %: how much of the fermentable sugar the yeast will eat. US-05 is typically around 81%. Higher attenuation = drier beer and higher ABV.
- If you're using multiple yeast strains, Brewski uses the first one's attenuation for the FG estimate.
- Lab and Product ID fields: optional, but helpful for tracking. If your yeast has a specific product code, put it here.
- Amount: how much yeast you're pitching. Not used in calculations but useful for your own records. See [Pitch rate](/reference/calculators/pitch-rate) if you want to calculate how much to pitch.

- [ ] **Step 5: Write `mash.md`**

Content to cover:
- The Mash tab lets you define your mash profile: steps, temperatures, times, and water ratios.
- A single-infusion mash is the default and works great for most styles. One step at a typical mash temperature (65–68°C / 149–154°F) for 60 minutes.
- Step mashing: hit **Add Step** to add more steps. Useful for styles that benefit from protein rests or mashout.
- Key fields per step: Step name, Type (Infusion, Temperature, Decoction), Target temperature, Duration (minutes), Infuse amount (if applicable).
- Water:grain ratio: how much water you add per kg (or lb) of grain. Thicker mashes (lower ratio) generally give better efficiency; thinner mashes are easier to stir and recirculate. 3 L/kg is a common starting point for BIAB.
- Strike temperature: Brewski calculates the water temperature you need to hit your mash temp target, accounting for grain absorption and your equipment's thermal mass. You'll see it in the stats sidebar once you have a mash step and a grain bill.
- Grain temperature: set this in the Mash tab. If your grain is at room temperature (20°C), leave it. If it's cold, adjust it — it affects the strike temp calculation.

- [ ] **Step 1: Write `water-chemistry.md`**

Content to cover:
- Water chemistry affects mash pH, flavor perception, and hop character. You don't have to use it, but it's there when you're ready.
- The Water tab on a recipe has two parts: water sources (what goes in) and mineral adjustments (what you add).
- Setting up water sources: add your tap water or filtered water as a source. You can set the ion levels (Ca, Mg, Na, Cl, SO₄, HCO₃) if you have a water report. If not, use a known profile like "Balanced" or "Soft."
- Mineral additions: gypsum, calcium chloride, baking soda, etc. Enter the grams you plan to add and Brewski shows you the resulting ion levels.
- Target profiles: if you're brewing to a style, there are common water profiles to aim for (e.g. Burton-on-Trent for English IPAs, Pilsen for lagers). Use these as guidance, not gospel.
- See [Water ions & their effects](/concepts/water-ions) for what each ion actually does to your beer.

- [ ] **Step 2: Write `scaling.md`**

Content to cover:
- Hit the scale icon (⤡) in the recipe header to open the Scale Recipe modal.
- Enter your new batch size. Brewski recalculates all ingredient amounts proportionally.
- Efficiency adjustment: if you're scaling significantly (e.g. 5 gal to 10 gal), your mash efficiency might change. You can adjust the target efficiency in the modal before applying.
- Scaling creates a new version of the recipe. The original is preserved in the version history.
- Hop amounts scale linearly. Note that bitterness perception can shift at very different scales — you may want to review IBU targets after a big scale-up.

- [ ] **Step 3: Write `recipe-versions.md`**

Content to cover:
- Every time you save a recipe, Brewski creates a version. You can view the full version history via the clock icon (🕐) in the recipe header.
- Versions are numbered sequentially. You can see what changed between versions (ingredient additions, amount tweaks, etc.).

- [ ] **Step 3: Write `refractometer.md`**

- **Post-fermentation (FG correction)**: Inputs: OG in Brix, FG in Brix (raw refractometer reading), wort correction factor. Output: corrected FG in SG.

- Wort correction factor (WCF): most wort reads slightly higher than pure sugar solutions. Default WCF is 1.04. Check your refractometer's documentation — some are pre-corrected.

- Brix: used primarily in winemaking and with refractometers. Numerically very close to Plato for wort.

## Task 9: Concepts & Glossary

**Files:**
- Write: all 9 files in `docs/site/concepts/`

- [ ] **Step 1: Write `gravity.md`**

Content:
- Gravity is a measure of how much dissolved sugar is in your wort. More sugar = higher gravity = more food for yeast = more alcohol potential.
- OG (Original Gravity): measured before fermentation. FG (Final Gravity): measured after.
- Water is 1.000. A session ale might have an OG around 1.040. A barleywine might be 1.100+.
- SG vs Plato vs Brix: all three measure the same thing, just on different scales. SG is most common in homebrewing. Plato is preferred commercially. Brix is used in winemaking and with refractometers.
- Gravity points: sometimes you'll see "40 gravity points" meaning 1.040. Makes the math easier when comparing recipes.

- [ ] **Step 2: Write `ibu.md`**

Content:
- IBU stands for International Bitterness Units. It's a measure of the alpha acids in your finished beer.
- Higher IBU = more potential bitterness, but perceived bitterness depends on balance. A 60 IBU DIPA tastes balanced if the OG is 1.080. A 60 IBU pale ale tastes intensely bitter.
- BU:GU ratio: IBU divided by (OG - 1) × 1000. A ratio around 1.0 is balanced. Higher = more bitter.
- Tinseth formula: Brewski uses the Tinseth IBU formula, which is the most widely used homebrewing model. It accounts for boil time, wort gravity, and hop alpha acids.
- Why your actual IBUs might differ: hop alpha % varies by harvest year, your kettle utilization might differ from the model, whole hops have lower utilization than pellets.

- [ ] **Step 3: Write `color.md`**

Content:
- Beer color is measured in SRM (US standard) or EBC (European standard). Higher number = darker beer.
- Morey formula: Brewski calculates color using the Morey formula, which is more accurate than older formulas for very dark beers.
- SRM ranges: 1–4 (pale lager), 5–8 (golden ale), 9–14 (amber/red), 15–22 (brown ale, porter), 23–40 (stout), 40+ (imperial stout).
- SRM ≈ EBC / 1.97. Lovibond is roughly equal to SRM in the homebrewing range.
- Color is an estimate. The actual color of your beer depends on wort clarity, fermentation, filtration, and glass shape.

- [ ] **Step 4: Write `abv-attenuation.md`**

Content:
- ABV is the percentage of alcohol by volume in your finished beer.
- It's calculated from the difference between OG and FG: `(OG - FG) × 131.25`.
- Apparent attenuation: the percentage of the original gravity that the yeast fermented. `(OG - FG) / (OG - 1.000) × 100`.
- Real attenuation is slightly lower than apparent because alcohol is less dense than water, making the FG reading appear lower than the true sugar content.
- Calorie estimation: most of a beer's calories come from alcohol (7 kcal/g) and residual carbohydrates (4 kcal/g). Higher ABV and higher FG = more calories.

- [ ] **Step 5: Write `mash-chemistry.md`**

Content:
- The mash converts grain starches into fermentable sugars. Two enzymes do the work: alpha-amylase (works better at 68–72°C) and beta-amylase (works better at 63–66°C).
- Mash temperature affects fermentability: lower temps (65°C) favor beta-amylase, giving a drier, more fermentable wort. Higher temps (68°C+) give a fuller body and more residual sweetness.
- Mash pH: target 5.2–5.4 for most styles. Outside that range, enzyme activity drops and you'll extract more harsh tannins.
- Efficiency: how much of the available sugar you extract from your grain. Typical homebrewing efficiency is 65–80% depending on your system and crush.
- Strike temperature: the water temperature needed to hit your target mash temp, accounting for the grain absorbing heat. Brewski calculates this automatically.

- [ ] **Step 6: Write `water-ions.md`**

Content:
- Six ions have the most impact on brewing water: calcium, magnesium, sodium, chloride, sulfate, and bicarbonate.
- **Calcium (Ca)**: promotes enzyme activity, yeast health, and protein precipitation. Target 50–150 ppm for most styles.
- **Magnesium (Mg)**: yeast nutrient at low levels. Harsh and laxative at high levels. Keep under 30 ppm.
- **Sodium (Na)**: enhances roundness and fullness at low levels (under 150 ppm). Salty or harsh above that.
- **Chloride (Cl)**: accentuates malt character and fullness. Higher Cl:SO₄ ratio = maltier-seeming beer.
- **Sulfate (SO₄)**: accentuates hop dryness and bitterness. Higher SO₄:Cl ratio = drier, more bitter-seeming beer.
- **Bicarbonate (HCO₃)**: raises mash pH. High bicarbonate is useful for dark malts that lower pH. For pale beers, keep it low.

- [ ] **Step 7: Write `hop-forms.md`**

Content:
- **Pellets (T-90)**: the most common form. Compressed and stabilized. Good shelf life, high utilization, easy to measure. Default assumption in Brewski's IBU calculation.
- **Whole/leaf hops**: less common, lower utilization than pellets, but some brewers prefer them for dry hopping. Absorb more wort volume.
- **Cryo hops (T-90 Cryo / LupuLN2)**: concentrated lupulin powder. Very high alpha and oil content — use roughly half the weight of regular pellets for the same bitterness. Set form to "Cryo" in Brewski.
- **T-45**: between T-90 and cryo in concentration.
- Utilization differences: Brewski adjusts IBU calculations based on hop form. Whole hops have ~10% lower utilization than pellets; cryo hops have ~50% higher concentration than T-90.

- [ ] **Step 8: Write `pitch-rate-starters.md`**

Content:
- Pitch rate: how many yeast cells you add per mL of wort per degree Plato. Standard rates: 0.75 M/mL/°P for ales, 1.5 for lagers.
- Under-pitching leads to stressed yeast: off-flavors, incomplete fermentation, slow starts.
- Over-pitching is usually fine for homebrewing, but extreme over-pitching can suppress some yeast character (important for Belgian styles).
- Yeast viability: liquid yeast packs lose viability over time. A fresh pack (~100B cells) at 4 weeks old might be 70% viable. The Pitch Rate calculator accounts for this.
- Starters: if your pack doesn't have enough viable cells, grow more before brew day. A 1L starter on a stir plate for 24 hours roughly doubles your cell count. The calculator tells you the starter volume needed.
- Dry yeast: most dry yeast packets contain 200B+ cells at high viability. For most homebrew batch sizes, one 11g packet is enough without a starter.
