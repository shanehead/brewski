A — Bottom Tab Bar

Persistent tabs at the bottom of the screen. Each tab is a full-screen section. Feels native on iOS and Android.

Pros

  • Native feel on both platforms
  • Always-visible navigation
  • Thumb-friendly

Cons

  • Limited to ~4-5 sections
  • Biggest change from desktop layout

B — Stack Navigation

List → Detail drill-down. Each screen fills the display. Hamburger or top-left icon opens a section switcher.

Pros

  • More screen space per view
  • Familiar mobile pattern

Cons

  • Requires more taps to navigate
  • Section switching less obvious

C — Adaptive (Phone + Tablet)

Bottom tabs on phones, sidebar panel on tablets. The layout adapts based on screen width breakpoints.

Pros

  • Best experience per device
  • Tablet closer to desktop

Cons

  • Most work to implement
  • Two navigation modes to maintain
